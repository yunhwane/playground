## List
- 동일한 특성의 데이터를 묶는 집합
- for문으로 데이터를 모두 접근이 가능하다.
- 데이터 중복을 허용한다.
- 리스트의 길이가 가변적이다. 이를 동적할당이라고 한다.
- 데이터가 연속적으로 나열된다.
- 데이터 사이의 빈공간이 없다.

### 장점
- 포인터(주소)로 데이터가 연결되어 있어, 연결 주소만 바꿔주면 되기때문에 데이터의 추가, 삭제가 용이하다.
- 메모리를 동적 할당 해주기 때문에 메모리 관리가 용이하다.

### 단점
- 객체를 다루기에 적은양의 데이터만 쓸 경우 차지하는 메모리가 커진다.

- 예를 들어 4byte 인 int, Wrapper class인 Integer는 32bit
객체 헤더 8byte+ 객체 주소 4byte + 객체 값 4byte = 최소 16byte를 차지한다.
- 기본적으로 순차적 할당이 아니기 때문에 검색 능력이 떨어진다.
